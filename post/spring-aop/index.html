Spring-Aop必知必会

2020-10-06
java后端

1. Spring-Aop简介

AOP的概念: AOP 为 Aspect Oriented Programming 的缩写，意思为面向切面编程，是通过预编译方式和运行期动态代理实现程序功能的统一维护的一种技术。利用AOP可以对业务逻辑的各个部分进行隔离，从而使得业务逻辑各部分之间的耦合度降低，提高程序的可重用性，同时提高了开发的效率。

AOP的作用: 作用：在程序运行期间，在不修改源码的情况下对方法进行功能增强。

AOP底层实现: AOP 的底层是通过 Spring 提供的的动态代理技术(JDK的动态代理 和 Cglib的动态代理)实现的。在运行期间，Spring通过动态代理技术动态的生成代理对象，代理对象方法执行时进行增强功能的介入，在去调用目标对象的方法，从而完成功能的增强。

2. AOP常用术语

Target（目标对象）：代理的目标对象

Proxy （代理）：一个类被 AOP 织入增强后，就产生一个结果代理类

Joinpoint（连接点）：所谓连接点是指那些被拦截到的点。在spring中,这些点指的是方法，因为spring只支持方法类型的连接点

Pointcut（切入点）：所谓切入点是指我们要对哪些 Joinpoint 进行拦截的定义

Advice（通知/ 增强）：所谓通知是指拦截到 Joinpoint 之后所要做的事情就是通知

Aspect（切面）：是切入点和通知（引介）的结合

Weaving（织入）：是指把增强应用到目标对象来创建新的代理对象的过程。spring采用动态代理织入，而AspectJ采用编译期织入和类装载期织入

@Component("myAspect")
@Aspect
public class MyAspect {
    /**
      切点表达式：execution([修饰符] 返回值类型 包名.类名.方法名(参数))

      几条规则：
      * 访问修饰符可以省略

      * 返回值类型、包名、类名、方法名可以使用星号*  代表任意

      * 包名与类名之间一个点 . 代表当前包下的类，两个点 .. 表示当前包及其子包下的类

      * 参数列表可以使用两个点 .. 表示任意个数，任意类型的参数列表

      execution(public void com.itheima.aop.Target.method())	
      execution(void com.itheima.aop.Target.*(..))
      execution(* com.itheima.aop.*.*(..))
      execution(* com.itheima.aop..*.*(..))
      execution(* *..*.*(..))
    */

    // 前置增强
    @Before("execution(切点表达式")
    public void before(){
        System.out.println("before...");
    }

    // 后置增强
    @AfterReturning("切点表达式")
    public void afterReturning(){}

    // 环绕增强--切入点方法执行前后都执行
    @Around("切点表达式")
    public void around(){}

    // 异常抛出通知--方法出现异常时执行
    @AfterThrowing
    public void afterThrowing(){}

    // 最终通知--必会执行，并且在所有切点和其他增强方法之后执行
    public void after(){}

}

4. 切点表达式抽取

抽取方式是在切面内定义方法，在该方法上使用@Pointcut注解定义切点表达式，然后在在增强注解中进行引用。具体如下：

@Component("myAspect")
@Aspect
public class MyAspect {
    @Before("MyAspect.myPoint()")
    public void before(){
        System.out.println("before...");
    }
    @Pointcut("切点表达式")
    public void myPoint(){}
}
